Announcement · SeqSieve v0.1.0

Announcing SeqSieve v0.1.0

A modern, fully client-side tool for exact sequence deduplication with provenance-preserving exports.

What is SeqSieve?

SeqSieve deduplicates FASTA and FASTQ records directly in the browser. It supports exact sequence-content deduplication, ID-based deduplication, full-record deduplication, normalization controls, reverse-complement-aware DNA/RNA matching, representative selection, and publication-ready reports.

SeqSieve performs exact deduplication. It is not a replacement for CD-HIT, MMseqs2, VSEARCH, or other approximate similarity-clustering tools when the goal is to group non-identical sequences by percent identity or coverage.

Why exact deduplication?

Exact deduplication is useful when redundant records are computational clutter rather than biological signal. Phage protein databases, microbial gene sets, HMM input collections, BLAST hit lists, and amplicon datasets often need cleanup before downstream analysis. SeqSieve keeps the context that explains where every sequence came from.

What is new in v0.1.0?

  • Robust FASTA parsing with multi-line records, duplicate-ID warnings, empty-sequence warnings, and line tracking.
  • FASTQ parsing with quality-length validation and mean Phred+33 scoring.
  • Deduplication by sequence content, sequence ID, or full record.
  • Normalization controls for case, whitespace, and gaps.
  • Optional reverse-complement-aware matching for DNA/RNA.
  • Exports for deduplicated FASTA/FASTQ, mapping TSV, duplicate groups TSV, counts TSV, summary TXT, reproducibility JSON, and methods text.

Tutorial 1: FASTA deduplication basics

  1. Open SeqSieve and select Try demo data, or upload test-data/simple_duplicates.fasta.
  2. Set format to Auto or FASTA.
  3. Set deduplication mode to By sequence content.
  4. Click Deduplicate and confirm 3 input records, 2 unique representatives, and 1 duplicate collapsed.
  5. Download the mapping, duplicate groups, counts, deduplicated FASTA, summary, and JSON files.

Tutorial 2: Reverse-complement-aware DNA cleanup

  1. Upload test-data/dna_reverse_complement_test.fasta.
  2. Set sequence type to DNA.
  3. Run with reverse complement off and confirm two unique records.
  4. Turn on Treat reverse complements as duplicates.
  5. Run again and confirm one unique representative with orientation recorded in the mapping table.

Tutorial 3: FASTQ quality-aware representatives

  1. Upload test-data/simple_fastq.fastq.
  2. Set format to FASTQ.
  3. Set representative rule to Highest mean quality for FASTQ.
  4. Run deduplication and inspect the mapping table to confirm the higher-quality read was selected.
  5. Keep the counts table with downstream notes because read deduplication can alter apparent abundance.

Tutorial 4: Warnings and malformed inputs

  1. Upload test-data/malformed_records.fasta.
  2. Run analysis and open the Warnings tab.
  3. Check text before the first header, empty records, duplicate IDs, and unexpected alphabet characters.
  4. Fix the source file where warnings are biologically meaningful, then rerun and export the clean report.

Reporting errors and requesting improvements

Useful reports include a small safe example, selected settings, expected output, observed output, browser and operating system, and whether the issue affects preview, export, parsing, or scientific interpretation. For feature requests, describe the biological workflow first and the desired UI second.

Hosting SeqSieve and this blog on GitHub Pages

  1. Create a new repository named SeqSieve.
  2. Add all files in this folder at the repository root.
  3. Commit with a message such as Release SeqSieve v0.1.0.
  4. Push to GitHub.
  5. Open repository settings, choose Pages, and publish from the root of the default branch.
  6. The app will live at https://mbaffour.github.io/SeqSieve/; the blog will live at https://mbaffour.github.io/SeqSieve/blog.html.
